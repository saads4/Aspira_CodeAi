📄 PRD — Real-Time Lab Tracking System (Next.js + MongoDB Edition)

Version: 2.1 (Stack-Specific)
Stack: Next.js 14 (App Router) + MongoDB + Redis
Architecture Style: Modular monolith (upgradeable to microservices)

1. 🎯 System Goal

Build a real-time lab tracking + dynamic ETA prediction system that:

Uses barcode ingestion
Integrates with SAP asynchronously
Calculates dynamic ETA (queue-aware)
Supports real-time updates (Zomato-style)
Handles batch scheduling + failures
Runs inside Next.js fullstack architecture
2. 🧠 Architecture Decision (IMPORTANT)

Since you're using Next.js:

👉 You are NOT building microservices initially
👉 You are building a modular monolith with clear boundaries

🏗️ Final Architecture
Next.js App
│
├── app/                → frontend (dashboard, scanner UI)
├── app/api/            → backend routes (Node runtime)
│
├── lib/
│   ├── db/             → MongoDB connection
│   ├── redis/          → Redis client
│   ├── events/         → event bus abstraction
│   ├── tat-engine/     → ETA logic
│   ├── scheduler/      → batch + delayed jobs
│   ├── sap/            → SAP integration
│
├── workers/            → background jobs (Node worker)
│
└── websocket-server/   → WS server (separate process)
3. 🗄️ Database Design (MongoDB Optimized)

fast reads (dashboard)
append-heavy writes (events)
denormalized structures

5. 🔄 Core Modules (Updated for Your Stack)
Module A — Barcode Ingestion API
Endpoint
POST /api/sample/ingest
Flow
1. Validate barcode
2. Check idempotency (scan_id)
3. Insert sample (Mongo)
4. Emit event
5. Trigger async SAP job
6. Return ETA
Idempotency (CRITICAL)

Store:

scanId

If exists → ignore duplicate

Module B — SAP Integration (Async Worker)
Implementation
Use BullMQ (Redis queue)
Flow
Scan → queue job → worker calls SAP
Retry Strategy
exponential backoff
max retries
fallback to EDOS
Module C — EDOS Rules Engine
Storage
Load Excel → JSON
Store in:
memory (fast)
Redis (shared)
Add this field (IMPORTANT):
labCapacity: number
Module D — TAT Engine (Node Module)
Function
computeETA(sample, batch, load, capacity)
Formula
queueDelay = currentLoad / capacity

ETA = batch.startTime 
    + queueDelay 
    + tat_hours
Trigger Points
new sample
batch change
processing complete

6. 🔁 Concurrency Control (MongoDB)

Use optimistic locking
