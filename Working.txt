System Workflow: Automated TAT & Batch Monitoring

Architecture: Pure Node.js (Express + BullMQ)
Flow: SAP Webhook → Entry Regulator → BullMQ (Redis) → Processing Engine (Worker) → MongoDB + Alerts

1) ENTRY REGULATOR (Node.js/Express)
- Endpoint: POST /webhook
- Responsibilities:
  - Receives SAP payload (supports single object or { "tests": [...] } array).
  - Validation: Ensures all required fields exist (no null/undefined).
  - Normalization: Creates unique records per test.
  - Caching: Stores transient record in Redis (`sample:{id}:{test}`) for 24h.
  - Queueing: Enqueues job to BullMQ `sample-processing` queue.
- Response: Returns HTTP 202 Accepted immediately.

2) QUEUE SYSTEM (BullMQ/Redis)
- Event-Driven: No database polling.
- Priority: "HIGH" priority samples (from SAP) are bumped to the front of the queue.
- Reliability: Jobs stay in Redis until acknowledged; retries on failure with exponential backoff.

3) PROCESSING ENGINE (Node.js Worker process)
- Event Source: Listens to BullMQ queue.
- EDOS Lookup (src/services/edosLoader.js):
  - Sub-millisecond lookup in memory (Map).
  - Loads 1070+ test rules from `Edos List.csv` (Fallback to `data/edos.json`).
- Batch Assignment (src/services/batchAssigner.js):
  - Converts received_at to Asia/Kolkata timezone (Luxon).
  - Scans for first valid batch window where `cutoff_time >= received_time`.
  - Scans up to 30 days forward to ensure NO sample remains unassigned.
- Computation:
  - ETA: Calculated based on Batch Run Start + TAT (Relative or Absolute).
  - SLA Deadline: Calculated as received_at + agreed_tat_hours.
  - Breach Detection: Flagged TRUE if ETA > SLA.
  - Overage: Computed as (ETA - SLA) in minutes.

4) STORAGE & DASHBOARD (MongoDB Atlas)
- Final Storage: Worker upserts results into the `samples` collection.
- Statuses: 
  - 'assigned': Batch found within day.
  - 'delayed': Missed original batch, moved to next available (High Priority).
  - 'error': Processing failed.
- Processed Flag: Set to true once worker completes logic.

5) ALERT SYSTEM (src/services/alertService.js)
- Persistent Logging: Every alert (MISSED_BATCH, SLA_BREACH, DELAY_ESCALATION) is saved in the `alerts` collection in MongoDB.
- Console Monitoring: Colorized terminal alerts (Red for Missed, Yellow for Breach, Magenta for Escalation).
- Emailing: Optional SMTP email with full sample snapshot (Times, Priority, Reason, Action).
- API Visibility: `GET /api/alerts` provides a full audit trail of recent system exceptions.

6) DASHBOARD API (node-service/src/routes/...)
- /api/samples: Dashboard main view (Search/Filter by status/breach).
- /api/stats: Aggregate counters (Assigned, Breached, Pending).
- /api/batches: Heatmap/utilization view of batch queues.
- /api/alerts: Audit log of system failures.

7) RESILIENCE & OPERATIONS
- Graceful Shutdown: Server and Worker handle SIGINT/TERM to finish jobs and close connections cleanly.
- Logging: Centralized logger with timestamps and ISO formatting.
- Tests: `npm test` runs 28+ assertions on core TAT/Batch logic without needing a database.
