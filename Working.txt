SAP System → Webhook Trigger → Backend Processing → Batch Assignment → ETA → Alerts → Dashboard

1)	Entry Regulator implement via Node js
- WebHook is triggered via a SAP api 
each SAP contain following payload
test_name
method
specimen_type
agreed_tat_hours
priority_tat
batch_windows
-here node validates the payload ensuring nothing is empty also make sure if there are multiple tests in the payload then it create one record per test
-then all this records  is written in the database mongo db atlas with the status pending or processed false
-return accepted message
2) Processing Engine implemented via python
- here python check the database every 5 seconds when it finds the status pending in the db,
- it pickup and starts processing 
- Python looks up the test code in edos.json. It retrieves the batch windows, agreed TAT hours, and priority TAT rules
- Python looks at the received_at timestamp, converts it to lab local time (Asia/Kolkata), then walks through the test's batch windows from earliest to latest. The first window whose cutoff time is at or after the received time is assigned. If all windows for today are missed, it rolls forward day by day until it finds the next valid window.
- ETA is Calculated = batch run start time + TAT minutes
-SLA Deadline	received_at  +  agreed_tat_hours
- SLA Breach	ETA  >  SLA Deadline  →  True
- Overage	(ETA  −  SLA Deadline)  ÷  60  =  minutes over
- MongoDB is updated
The sample document is updated with batch_id, batch_run_start, ETA, SLA deadline, breach flag, overage minutes, and status changed to 'assigned'. The processed flag is set to true so it is not picked up again.
- Finally a alerts 
Alerts are sent
If the batch was missed, a MISSED BATCH alert is fired. If the ETA breaches the SLA, an SLA BREACH alert is fired. Both are printed to the terminal in colour, and if Gmail is configured, an email is sent to the configured recipient.
Every alert is always printed to the Python terminal in colour so you can see it in real time. If email is configured in your .env file, the same alert is also sent as a formatted plain-text email to the configured recipient address.
Each alert email contains a complete summary:
•	Sample ID and test name
•	Priority level
•	When the sample was received
•	Which batch it was assigned to and when that batch runs
•	ETA and SLA deadline
•	Overage in minutes (for SLA breach alerts)
•	The reason for the missed batch (for missed batch alerts)
•	Recommended action
